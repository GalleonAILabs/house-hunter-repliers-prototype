Every process that runs the house search.

A mobile-first tool that lets a family shortlist homes together: shared ratings and notes, live GO-commute and mortgage math, and a map, all served from one small stdlib Python server behind a Cloudflare tunnel. This is the complete map of how it runs, end to end.

GET/api/feedbacktokenLatest rating/note/reject/research per person per listing

GET/api/household-settingstokenShared mortgage assumptions
GET/layers/*.geojsonopenGO stations, GO lines, Highway 413 overlays
GET/ · /app.js · /styles.cssopenThe static app itself
POST/api/feedbacktokenRecord a rating / note / reject / research, attributed to a person
POST/api/potential-purchase-pricestokenUpsert the shared price (must be positive)
POST/api/poitokenDrop a shared map pin
POST/api/household-settingstokenUpsert a shared assumption (by key)

* /api/config is deliberately open so the front end can bootstrap the token it then sends on every protected call. The token is a deterrent against a stranger finding the URL, not real security. See the Cloudflare Access item in TODOS.md for the planned gate.

TableShapeScopeNotes

Shared tables (price, settings, POI) hold one value for the whole buyer group; listing_feedback is the only per-person store. Every write stamps updated_by / person_id, so the app always knows who did what.
